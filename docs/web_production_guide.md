# Web制作におけるヒアリングシート項目と推奨技術ガイド

## 1. ヒアリングシート必須項目

Webサイト制作を成功させるために、以下の項目をクライアントから詳細に引き出す必要があります。

### A. プロジェクト概要・目的
- [ ] **サイト制作の目的**: 新規立ち上げ、リニューアル、集客強化、採用強化、ブランディングなど
- [ ] **ターゲット層 (ペルソナ)**: 年齢、性別、職業、悩み、サイト訪問時の心理状態
- [ ] **USP (Unique Selling Proposition)**: 競合他社に負けない自社の強み・特徴
- [ ] **ゴール (CV)**: 資料請求、問い合わせ、購入、会員登録、採用応募など

### B. デザイン・世界観
- [ ] **参考サイト**: 好みのデザインのURL（3つ程度）、理由（色使い、レイアウト、フォントなど）
- [ ] **NGサイト**: 嫌いなデザインのURL、理由
- [ ] **イメージカラー**: メインカラー、アクセントカラー、企業のロゴカラー
- [ ] **トンマナ**: 高級感、ポップ、信頼感、親しみやすさ、シンプル、未来的など

### C. コンテンツ・機能要件
- [ ] **ページ構成 (サイトマップ)**: TOP、会社概要、サービス紹介、実績、FAQ、問い合わせ、採用情報など
- [ ] **必要な機能**: 問い合わせフォーム、予約システム、会員ログイン、ブログ/ニュース更新、多言語対応、検索機能
- [ ] **素材の有無**: ロゴデータ（AI/PNG）、写真素材、原稿テキスト、動画素材
- [ ] **CMS (更新システム)**: クライアント側で更新が必要な箇所とその頻度

### D. 技術・インフラ要件
- [ ] **ドメイン**: 取得済みか、新規取得か
- [ ] **サーバー**: 契約済みか、推奨サーバーの手配が必要か
- [ ] **SEO対策**: 狙いたいキーワード、競合サイト
- [ ] **アクセス解析**: Google Analytics (GA4)、Search Console の導入希望
- [ ] **ブラウザ対応**: 必須対応範囲（PC/スマホ/タブレット、IE対応要否など）

### E. 予算・スケジュール・運用
- [ ] **予算感**: 松竹梅のプラン提案可否、保守費用の有無
- [ ] **希望納期**: サイト公開希望日、素材提供日
- [ ] **運用体制**: 公開後の更新担当者、保守契約の要否

## 2. おすすめの制作ツール・技術スタック

目的、予算、クライアントの運用体制に合わせて最適なツールを選定します。

### パターンA：『高品質・高パフォーマンス・ブランディング重視』
**推奨技術**: **Next.js (React) + Vercel / Netlify + Headless CMS (microCMS / Keystatic / Contentful)**

*   **概要**: モダンな技術構成により、圧倒的な表示速度と自由度の高いデザインを実現。
*   **おすすめのケース**:
    *   企業のコーポレートサイト、サービスサイト（LP）、キャンペーンサイト
    *   SEOで上位表示を狙いたい場合（Core Web Vitals対策が容易）
    *   複雑なアニメーションやアプリのような操作性を実装したい場合
    *   「NEXT VALLEY」の制作事業として強みを出せる領域
*   **メリット**:
    *   表示速度が非常に速く、ユーザー体験が良い
    *   セキュア（WordPressのような脆弱性リスクが低い）
    *   再利用可能なコンポーネント設計が可能
*   **デメリット**:
    *   開発コスト（工数・単価）が比較的高くなる
    *   更新機能はCMSを別途導入・設計する必要がある

### パターンB：『更新頻度が高い・中規模以上のメディア』
**推奨技術**: **WordPress (With Original Theme or High-Performance Themes like Snow Monkey/SWELL)**

*   **概要**: 世界シェアNo.1のCMS。プラグインが豊富で、ブログ機能や会員機能などを低コストで実装可能。
*   **おすすめのケース**:
    *   オウンドメディア、ブログ記事がメインのサイト
    *   クライアントがWordPressの操作に慣れている場合
    *   多機能なプラグイン（予約、ECなど）を安価に使いたい場合
*   **メリット**:
    *   情報が多く、開発・運用コストを抑えやすい
    *   クライアント側での更新作業が容易
*   **デメリット**:
    *   表示速度の最適化に手間がかかる（キャッシュ制御など）
    *   セキュリティメンテナンス（アップデート）が必須
    *   デザインのカスタマイズがテーマ依存になりがち

### パターンC：『低予算・短納期・デザイン重視』
**推奨技術**: **Studio / Framer / Webflow (No-Code Tools)**

*   **概要**: コーディング不要でデザインをそのままWebサイトにできるツール。
*   **おすすめのケース**:
    *   ランディングページ（LP）、簡単なコーポレートサイト
    *   とにかく早く公開したい場合（最短数日〜）
    *   予算を最小限に抑えたいスタートアップや個人事業主
*   **メリット**:
    *   **爆速**で納品可能
    *   サーバー設定などの保守運用がほぼ不要
    *   デザイン修正が直感的に行える
*   **デメリット**:
    *   複雑な機能（高度なフォーム、会員機能など）の実装が難しい
    *   プラットフォーム利用料が月額で発生する
    *   大規模なデータ移行や拡張性に制限がある

## 結論：AI×Web制作事業としての推奨
「AI制作」を掲げるWeb制作事業であれば、**Next.js** をメインの武器にすることをお勧めします。理由は以下の通りです。

1.  **AIとの相性が良い**: 生成AIを活用したコード生成や、AIチャットボットの組み込み（Vercel AI SDKなど）が容易。
2.  **差別化**: WordPressやNo-Code制作会社は飽和していますが、Next.jsで高品質なサイトを作れる会社はまだ価値が高いです。
3.  **実績**: 自社サイト（NEXT VALLEY）もNext.jsで構築されており、ノウハウが蓄積されています。

一方で、予算重視のクライアント向けには **Studio** 制作プランを用意しておくと、取りこぼしを防げます。

## 3. ヒアリングシート作成のおすすめツール

Web制作事業では「クライアント体験」も重要です。「AI×Web制作」を名乗る以上、WordやExcelファイルを送るのではなく、以下のデジタルツールでのヒアリングを推奨します。

### 推奨1：Notion（ノーション）
*   **おすすめ**: ブランディングを重視し、クライアントと共同編集したい場合。
*   **メリット**:
    *   **「今っぽい」印象**: AIやDXに強いイメージを与えられます。
    *   **自由度**: 画像や参考URLの貼り付けが簡単で、コメント機能でその場で議論できます。
    *   **管理**: 議事録としても機能し、プロジェクト管理ページとしてそのまま使えます。
*   **配布方法**: テンプレートページを作成し、クライアントをゲスト招待して直接記入してもらうか、複製してもらいます。

### 推奨2：Google Forms / Typeform
*   **おすすめ**: 手軽さ重視。初回ヒアリングを効率化したい場合。
*   **メリット**:
    *   **簡単**: URLを送るだけで、スマホからでも回答可能。
    *   **集計**: 回答が自動でスプレッドシートにまとまるため、顧客管理が楽です。
*   **Typeform**: デザイン性が高く、一問一答形式で回答率が高いツールです（有料プラン推奨）。

### 推奨3：自社サイトへの実装（Next.js の活用）
*   **おすすめ**: **技術力をアピールしたい場合（最強のポートフォリオ）。**
*   **メリット**:
    *   **実力証明**: 「自社のヒアリングフォームもここまで作り込んでいます」というアピールになります。
    *   **完全カスタマイズ**: 独自のバリデーションや、AIによる自動要約などを組み込めます。
*   **実装案**:
    *   複数ステップのフォーム（Wizard形式）にする。
    *   shadcn/ui の Form コンポーネントなどを活用してモダンなUIにする。

## 5. サーバー・ドメインの契約・管理方針

**結論：必ずクライアント自身に契約させてください（クレカ払い推奨）。**

Web制作事業において、ここを曖昧にすると後々トラブルの原因になります。
以下の理由から、**「制作側による代理契約・立て替え」は避けるべき** です。

### 理由1：所有権の明確化
*   **ドメイン（.comなど）**: 企業の「資産」です。制作会社が契約していると、契約終了時や会社がなくなった時にドメインを失うリスクがあります。
*   **サーバー**: データ（顧客情報など含む）の置き場所です。法的な責任の所在を明確にするためにも、クライアント名義が必須です。

### 理由2：金銭トラブルの回避
*   **更新忘れ**: 「クライアントからの入金がなく、制作側が更新料を立て替えるか悩んでサーバー停止」という事例が頻発します。
*   **値上げ**: サーバー代が高騰した際、保守費用に上乗せするための交渉が面倒になります。

### 理由3：Googleなどの規約（特にVercel）
*   **Next.js (Vercel)**: 商用利用の場合、Proプラン($20/月)が必要になるケースがあります。これを代理契約すると、規約違反やアカウント停止のリスクがあります。クライアントのチームに招待してもらう形が安全です。

### 運用フローの推奨
1.  **サポート**: 「おすすめのサーバー」や「ドメイン会社」をチャットで教える。
2.  **Zoom画面共有**: 不安なクライアントには、Zoomを繋ぎながら一緒に契約画面を操作してあげる（ここまでやれば親切）。
3.  **情報共有**: 契約完了後に、ログインID/PWを共有してもらう（Notion等で安全に管理）。

3.  **情報共有**: 契約完了後に、ログインID/PWを共有してもらう（Notion等で安全に管理）。

※ **例外**: 「月額◯万円の保守管理プラン」として、サーバー代込みで請求する場合のみ代理契約もありですが、解約時の移管手続きが非常に手間なので、やはり非推奨です。

## 6. Vercel と GitHub の契約・運用フロー詳細

「クライアントにどこまでやってもらうか？」という質問への回答です。
**結論：クライアントは「支払い情報（クレカ）の登録」だけ。あとは全て制作側が代行します。**

### A. GitHub（プログラムコードの管理）
*   **契約者**: **制作会社（あなた）**
*   **理由**: クライアントはコードを見ても分からないため、GitHubアカウントを作らせる必要はありません。
*   **納品形態**:
    *   通常は「ビルドされた成果物（Webサイト）」が納品物です。
    *   コード自体の譲渡契約がない限り、リポジトリはあなたのGitHubアカウント内で管理し、クライアントには見せなくてOKです。

### B. Vercel（サーバー・公開環境）
*   **契約者**: **クライアント（推奨） or 代理契約**
*   **理由**: 商用利用のProプラン（$20/月）の支払いをクライアントカードで行うため。
*   **推奨フロー（一番スムーズな方法）**:
    1.  **アカウント作成**: クライアントに「Vercelアカウント」を作成してもらう（メールアドレスでOK）。
    2.  **チーム作成**: Vercel上で「Team」を作成し、クレジットカードを登録してもらう。
    3.  **招待**: そのTeamに、制作会社（あなた）のアカウントを `Member` または `Owner` として招待してもらう。
    4.  **開発・連携**: あなたが自分のGitHubリポジトリを、クライアントのVercel Teamに接続してデプロイ設定を行う。
    
    これにより、**「コード管理はあなた（GitHub）」「サーバー支払いはクライアント（Vercel）」** という綺麗な分担ができます。クライアントは何も操作する必要がありません。

### C. ドメイン管理（お名前.com / Xserver Domainなど）
*   **契約者**: **クライアント**
*   **作業**:
    1.  クライアントに取得してもらう。
    2.  DNS設定（Vercelに向ける設定）だけ、ログイン情報を一時的に借りてあなたが設定代行する。
### C. ドメイン管理（お名前.com / Xserver Domainなど）
*   **契約者**: **クライアント**
*   **作業**:
    1.  クライアントに取得してもらう。
    2.  DNS設定（Vercelに向ける設定）だけ、ログイン情報を一時的に借りてあなたが設定代行する。

### D. 別案：自社のVercelアカウントに載せる場合（エージェンシーモデル）
「保守費として月額をもらい、自社のVercelアカウントで全部運用する」という方法も **技術的には可能で、よく行われています。**

*   **メリット**:
    *   クライアントの手間がゼロ（クレカ登録すら不要）。
    *   **「保守管理費」としての収益化** がしやすい（例：月額1万円でサーバー・ドメイン管理込み）。
*   **デメリット・リスク（※重要）**:
    *   **アカウント停止リスク**: 1つのクライアントが規約違反やDDoS攻撃を受けると、**あなたのアカウント全体（他クライアント含む）** が停止されるリスクがあります。
    *   **コスト負担**: Vercelの帯域制限（Bandwidth）などを超過した場合、あなたが追加料金を払う必要があります。
    *   **契約解除時の手間**: クライアントが「自社で管理したい」と言い出した時の移行手続き（Transfer）が少し面倒です（相手にProプラン契約を求める必要がある）。

**結論**:
*   個人商店レベルの小規模サイトなら「自社Vercelへの同居」もアリですが、
*   将来的にスケールさせるなら、最初から「クライアントごとのTeam作成」を推奨します（リスク分散のため）。

## 7. プレビュー制作（無料診断）前の重要ヒアリング項目

「無料プレビュー（デモサイト）」を作る段階では、全項目を聞く必要はありません。
**「相手に刺さるデザイン」と「的外れでないコピー」を作るための最小限の情報（5つ）** だけを聞いてください。

### ①【必須】ビジネスの強み・売り（USP）一言で言うと？
*   **目的**: ヒーローセクション（トップ画像のキャッチコピー）を作るため。
*   **質問例**: 「競合他社に負けない一番の強みは何ですか？」「お客様に一番伝えたい言葉は何ですか？」
*   **これがないと**: 綺麗なだけで中身のないサイトになり、響きません。

### ②【必須】ターゲット層（ペルソナ）は誰か？
*   **目的**: デザインの方向性（高級感 vs 親しみやすさ、文字サイズなど）を決めるため。
*   **質問例**: 「メインのお客様はどんな方ですか？（30代女性、法人企業、高齢者など）」
*   **これがないと**: 例えば「高齢者向けなのに文字が小さくてお洒落すぎるサイト」を作ってしまい、逆効果になります。

### ③【必須】参考にしたいサイト（デザインの好み）× 2〜3個
*   **目的**: クライアントの頭の中にある「いい感じ」を具体化するため。
*   **質問例**: 「同業種でも異業種でも構いません。デザインの雰囲気が好きなサイトのURLを2〜3個教えてください。」
*   **これがないと**: 「イメージと違う」と言われて作り直しになるリスクが激増します。

### ④【任意】競合他社・ベンチマークのサイト
*   **目的**: 競合より「少しだけ良い」クオリティで作ることで、優位性を示すため。
*   **質問例**: 「一番意識しているライバル企業、または目標としている企業のサイトはありますか？」
*   **これがないと**: 業界の標準レベルが分からず、提案のレベル感を外す可能性があります。

### ⑤【任意】現在のロゴデータ・写真素材（あれば）
*   **目的**: サイトの配色をロゴに合わせるため。写真はAI生成でも仮置きできますが、リアルな写真が1枚でもあると説得力が増します。
*   **質問例**: 「ロゴデータや、使ってほしい写真はありますか？なければこちらで仮画像を用意します。」

**戦略**:
この5つさえあれば、**「構成案（ワイヤーフレーム）」を飛ばしていきなり「高クオリティなトップページ」を見せる** ことができます。これが「AI×Web制作」の最大の武器（スピード）です。

## 8. LINEリッチメニューのおすすめ項目

Web制作事業の公式LINEであれば、**「無料プレビュー（デモ制作）への誘導」を最優先** にしつつ、信頼感を担保する構成がベストです。

### おすすめ構成（6分割タイプ）

スマホ画面の下半分を占有するため、視認性が抜群です。

| 上段左（一番目立つ） | 上段中 | 上段右 |
| :--- | :--- | :--- |
| **無料プレビューを作る**<br>（🔗 ヒアリングシートへ） | **制作実績を見る**<br>（🔗 サイトのWorksへ） | **料金プラン**<br>（🔗 サイトのPriceへ） |

| 下段左 | 下段中 | 下段右 |
| :--- | :--- | :--- |
| **制作の流れ**<br>（🔗 サイトのProcessへ） | **よくある質問**<br>（🔗 サイトのFAQへ） | **チャットで相談**<br>（テキスト入力モード切替） |

### 各項目のポイント

1.  **無料プレビュー（CVエリア）**:
    *   **色**: ここだけ目立つ色（ブランドカラーの濃い色やオレンジなど）にします。
